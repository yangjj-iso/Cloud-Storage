# 模块七：异步校验与转码

## 一、为什么要异步？

用大白话讲：

- **视频转码耗时数分钟**：用户上传完要等很久，体验极差，同步做 TP99 直接爆炸
- **合并 50GB 文件后计算 MD5 需要数分钟**：不能阻塞用户返回，用户等不起
- **转码失败不应影响文件可用性**：文件已经可用，转码只影响缩略图/转码产物，不能因为转码挂了就把文件也搞废了
- **用 RocketMQ 解耦**：上传链路 TP99 ≤ 200ms，用户秒传完，后面的事慢慢干

## 二、异步校验：Checksum

### 2.1 流程

合并成功后投递 RocketMQ 消息到 `cloudchunk-checksum` topic，`ChecksumConsumer` 消费后做整文件 MD5 校验。

### 2.2 核心代码

```java
// ChecksumConsumer.java —— MQ 消费入口
@RocketMQMessageListener(topic = MqTopics.CHECKSUM, consumerGroup = MqTopics.CG_CHECKSUM,
        consumeMode = ConsumeMode.CONCURRENTLY)
public class ChecksumConsumer implements RocketMQListener<ChecksumMessage> {
    @Override
    public void onMessage(ChecksumMessage msg) {
        TraceContext.set(msg.getTraceId());   // 链路追踪透传
        try {
            service.verify(msg);
        } catch (Exception e) {
            throw e;  // 抛出 → RocketMQ 重投
        } finally {
            TraceContext.clear();
        }
    }
}
```

```java
// ChecksumService.java —— 实际校验逻辑
public void verify(ChecksumMessage msg) {
    FileMeta meta = fileMetaService.findById(msg.getFileId()).orElse(null);
    if (meta != null && meta.getStatus() == FileStatus.AVAILABLE) return;  // 幂等

    // 流式 MD5（不加载整文件到内存）
    String actual;
    try (InputStream in = storageFactory.current()
            .get(new GetRequest(msg.getBucket(), msg.getObjectKey()))) {
        actual = Md5Utils.md5(in);   // 内部 DigestInputStream + 1MB buffer
    }

    if (!actual.equalsIgnoreCase(msg.getExpectMd5())) {
        fileMetaService.updateStatus(msg.getFileId(), FileStatus.BROKEN);
        brokenProducer.publish(BrokenMessage.of(msg.getFileId(), "md5 mismatch"));
        storageFactory.current().delete(msg.getBucket(), msg.getObjectKey());
        return;
    }

    fileMetaService.updateStatus(msg.getFileId(), FileStatus.AVAILABLE);
    transcodeProducer.publish(t);  // 投递转码消息
}
```

### 2.3 用大白话解释

1. **幂等**：如果已经是 AVAILABLE 状态，直接返回，不重复校验。防重试消息重复处理。
2. **流式 MD5**：用 `DigestInputStream` + 1MB buffer，边读边算，不加载整文件到内存。50GB 文件也能轻松算，内存不会爆。
3. **校验通过**：置 AVAILABLE + 投递转码消息，文件正式可用，转码交给下游。
4. **校验失败**：置 BROKEN + 删除对象 + 投递损坏通知，文件坏了，不能留，必须清理。

### 2.4 流式 MD5 代码

```java
try (InputStream in = storage.download(bucket, finalKey);
     DigestInputStream dis = new DigestInputStream(in, MessageDigest.getInstance("MD5"))) {
    byte[] buf = new byte[1024 * 1024]; // 1 MB buffer
    while (dis.read(buf) != -1) { /* just read */ }
    byte[] digest = dis.getMessageDigest().digest();
    return Hex.encodeHexString(digest);
}
```

## 三、异步转码：模板方法 + TAG 路由

### 3.1 流程

Checksum 通过后 → `TranscodeProducer` 按 MIME Type 分发 → img/video/doc Worker

### 3.2 RocketMQ Topic / Tag 规范

| Topic | Tag | Consumer Group | 说明 |
|---|---|---|---|
| cloudchunk-checksum | — | CG-checksum | 整文件 MD5 校验 |
| cloudchunk-transcode | img | CG-transcode-img | 图片转码 |
| cloudchunk-transcode | video | CG-transcode-video | 视频转码 |
| cloudchunk-transcode | doc | CG-transcode-doc | 文档转码 |

### 3.3 模板方法模式

```java
// AbstractTranscodeConsumer.java —— 模板方法模式
public abstract class AbstractTranscodeConsumer implements RocketMQListener<TranscodeMessage> {

    protected abstract String taskType();
    protected abstract TranscodeResult doTranscode(TranscodeMessage msg) throws Exception;

    @Override
    public void onMessage(TranscodeMessage msg) {
        String doneKey = RedisKeys.transcodeDone(msg.getFileId(), taskType());
        if (Boolean.TRUE.equals(redis.hasKey(doneKey))) return;  // 幂等

        fileMetaService.updateTranscodeStatus(msg.getFileId(), TranscodeStatus.RUNNING);
        try {
            TranscodeResult result = doTranscode(msg);  // 子类实现
            fileMetaService.updateExtra(msg.getFileId(), extraJson, result.getThumbnailUrl());
            fileMetaService.updateTranscodeStatus(msg.getFileId(), TranscodeStatus.SUCCESS);
            redis.set(doneKey, "1", Duration.ofDays(7));
        } catch (TranscodeException te) {
            if (te.isRetryable()) throw te;  // 可重试 → RocketMQ 重投
            fileMetaService.updateTranscodeStatus(msg.getFileId(), TranscodeStatus.FAILED);
        }
    }
}
```

### 3.5 视频转码：FFmpeg

```java
@RocketMQMessageListener(topic = MqTopics.TRANSCODE, selectorExpression = MqTopics.TAG_VIDEO,
        consumerGroup = MqTopics.CG_TRANSCODE_VIDEO, consumeThreadNumber = 2)
public class VideoTranscodeConsumer extends AbstractTranscodeConsumer {
    @Override protected String taskType() { return "video"; }

    @Override
    protected TranscodeResult doTranscode(TranscodeMessage msg) throws Exception {
        // 1. 下载到本地临时文件
        // 2. FFmpeg 截取第 1 秒作为封面
        ffmpeg.run(List.of("-y", "-i", src.toString(),
                "-ss", "00:00:01", "-frames:v", "1", "-q:v", "3", cover.toString()));
        // 3. 转码 H.264
        ffmpeg.run(List.of("-y", "-i", src.toString(),
                "-c:v", "libx264", "-preset", "medium", "-crf", "23",
                "-c:a", "aac", "-b:a", "128k", "-movflags", "+faststart", dst.toString()));
        // 4. 上传产物
        storageFactory.current().put(PutRequest.of(..., coverKey, ...));
        storageFactory.current().put(PutRequest.of(..., outKey, ...));
        return TranscodeResult.video()...;
    }
}
```

### 3.6 重试与死信

- **可重试异常** → 抛出 → RocketMQ 重投（最多 3 次）
- **不可重试异常** → 标记 FAILED，不重试
- **超过 3 次进死信队列** → 告警 + 管理员可手动重试

### 3.7 转码幂等

用 Redis key：`cc:transcode:done:{fileId}:{type}`，SET EX 7 天

## 五、面试追问

| 问题 | 回答要点 |
|---|---|
| 为什么异步校验？ | 合并 50GB 文件后计算 MD5 需要数分钟，不能阻塞用户返回 |
| 为什么异步转码？ | 视频转码耗时数分钟，同步做 TP99 爆炸；转码失败不影响文件可用性 |
| 模板方法模式？ | AbstractTranscodeConsumer 定义 onMessage 模板，子类实现 doTranscode |
| 为什么用 TAG 分发？ | 同 Topic 按 selectorExpression 路由，图片 8 线程、视频 2 线程（CPU 密集） |
| 重试策略？ | 可重试 → 抛出让 MQ 重投；不可重试 → 标记 FAILED |
| 转码失败影响？ | 不影响，文件已 AVAILABLE |
| 转码幂等？ | cc:transcode:done:{fileId}:{type} SET EX 7 天 |
